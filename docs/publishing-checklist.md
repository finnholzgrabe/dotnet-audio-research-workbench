# Publishing Checklist

Before publishing:

- README quick-start commands work from a clean checkout.
- `dotnet build` passes.
- `dotnet test` passes.
- CI is green.
- No large generated artifacts are committed.
- No personal recordings are committed.
- External dataset licenses are documented.
- Medical disclaimer is visible from README.
- Claims in README match implemented features.
- Repository topics are selected on GitHub.

Suggested GitHub topics:

- `audio-processing`
- `signal-processing`
- `dotnet`
- `machine-learning`
- `research-software`
- `dsp`
